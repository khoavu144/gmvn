CRASH · React Rules of Hooks

GymDetailPage.tsx — useState sau early return (vi phạm Rules of Hooks)

Line 67: const [canReview, setCanReview] = useState(false) được khai báo sau 2 lệnh return sớm. React yêu cầu Hooks luôn gọi ở top của component, trước bất kỳ điều kiện hoặc return nào. Lỗi này sẽ crash app khi gym không load được.

// GymDetailPage.tsx — HIỆN TẠI (SAI) const [gym, setGym] = useState(null); const [loading, setLoading] = useState(true); ... if (loading) return <Spinner />; // ← early return if (!gym) return <NotFound />; // ← early return const [canReview, setCanReview] = useState(false); // ← SAI: Hooks sau return!
// FIX: Kéo tất cả useState lên top const [gym, setGym] = useState(null); const [loading, setLoading] = useState(true); const [canReview, setCanReview] = useState(false); // ← lên đây ... if (loading) return <Spinner />; if (!gym) return <NotFound />;

DATA MISMATCH · Type ≠ Entity

GymBranch — frontend type có fields không tồn tại trong DB entity

TypeScript interface GymBranch trong types/index.ts khai báo description và manager_name, nhưng entity GymBranch.ts không có 2 field này. Kết quả: API trả về null, UI cố đọc → undefined error.

// types/index.ts — có nhưng DB không có interface GymBranch { description: string | null; // ← không có trong entity manager_name: string | null; // ← không có trong entity }
// FIX A: Thêm vào entity GymBranch.ts @Column({ type: 'text', nullable: true }) description!: string | null; @Column({ type: 'varchar', length: 100, nullable: true }) manager_name!: string | null; // FIX B (đơn giản hơn): Xóa khỏi type nếu không cần dùng

UX

GymDetailPage — Gallery chỉ hiển thị 2 ảnh đầu

branchDetail.gallery.slice(0, 2) cắt gallery còn 2 ảnh. Không có nút "xem tất cả" hay lightbox. Gym có nhiều ảnh nhưng user không thấy.

Thêm grid gallery 4–6 ảnh + lightbox. Hoặc layout: 1 ảnh lớn bên trái + 2x2 ảnh nhỏ bên phải (Instagram-style), click mở full gallery.

ARCHITECTURE

Duplicate routes Coach — /coaches/:id vs /trainer/:id

Có 2 route riêng biệt cho cùng mục đích xem profile coach công khai: CoachDetailPage (tích hợp program, payment, gym badges) và ProfilePublic (slug-based, rich profile). Gây confusion SEO và UX.

Đề xuất: Giữ /coaches/:trainerId (CoachDetailPage) cho listing + subscription flow. Route /trainer/:trainerId redirect sang /coaches/:trainerId. Hoặc merge ProfilePublic vào CoachDetailPage. Không nên tồn tại song song 2 trang.

UX

Coaches.tsx — Không có filter chuyên môn / giá / thành phố

Trang danh sách Coach chỉ có ô text search. Không có filter specialty, price range, city, hay sort by (rating, giá, review). Backend trainerService đã có pagination nhưng chưa expose các params filter.

Thêm filter bar: Chuyên môn (multi-select), Giá/tháng (slider range), Thành phố (dropdown provinces.ts đã có!), Sắp xếp (rating, giá tăng/giảm, mới nhất). Backend thêm query params tương ứng.

PERFORMANCE

FeaturedCoaches — Random shuffle mỗi lần render

[...coaches].sort(() => 0.5 - Math.random()) trong useEffect không đảm bảo thứ tự ổn định, và shuffle lại mỗi khi component re-render. Không dùng useMemo.

// FIX: shuffle một lần, cache kết quả const shuffled = useMemo(() => { return [...data].sort(() => 0.5 - Math.random()).slice(0, 10); }, [data]); // chỉ tính lại khi data thay đổi
SEO

Không có dynamic <title> và meta tags

Toàn bộ SPA render cùng 1 title "GYMERVIET - Verified Fitness Profile". Trang chi tiết Coach, Gym không có Open Graph. Google chỉ thấy shell HTML trống.

Dùng react-helmet-async để set title/meta động per-page. Hoặc nếu muốn SEO tốt hơn: chuyển sang SSR (Remix/Next.js) cho các trang public. Tối thiểu: thêm sitemap.xml và robots.txt.

MAINTENANCE

AdminService.approveHighRiskAction — TODO chưa implement

Method approveHighRiskAction update DB log thành "approved" nhưng không thực sự execute action. Comment "TODO: Dispatch event" bỏ ngỏ. Admin click "Duyệt" nhưng action không xảy ra.

Cần implement dispatch thực sự hoặc dùng strategy pattern map action string → function. Hiện tại đây là dead code nguy hiểm (admin nghĩ đã approve nhưng thực ra không).

TECH DEBT

Profile.tsx — 987 dòng, quá lớn để maintain

File duy nhất 987 dòng chứa toàn bộ profile editor của trainer. Khó debug, khó test, mỗi PR thay đổi ở đây đều risk cao.

Split thành: ProfileBasicInfo.tsx, ProfileTrainerDetails.tsx, ProfileGallery.tsx, ProfileExperience.tsx, ProfileFAQ.tsx. Mỗi sub-component tự fetch data, dùng React Query. Profile.tsx chỉ giữ layout + tab navigation.

⚪ Nên bỏ / đơn giản hóa

4 thứ đang làm phức tạp codebase không cần thiết

✕ RevenueTier + Commission System

Entity RevenueTier implement hệ thống chia doanh thu 85/90/95% dựa trên monthly sales threshold — nhưng platform hiện đang build user base, chưa phải giai đoạn monetize phức tạp. Đây là over-engineering cho stage hiện tại.

→ Giữ FinancialTransaction để track, nhưng đơn giản hóa split thành cố định (ví dụ: 95% mãi) cho đến khi có đủ volume để negotiate tier.

✕ RankingCache Cron + Monthly Decay

Hệ thống gamification với điểm, leaderboard, 5% monthly decay, seasonal reset mỗi tháng. Cron job chạy ngầm, khó debug, ít user sẽ thấy trong giai đoạn đầu.

→ Giữ UserPointsHistory để thu thập data, nhưng tắt cron job và leaderboard UI đến khi platform đạt 500+ active users.

✕ AdminService High-Risk Approval Flow

Hệ thống 2-admin approval cho high-risk actions (không thể tự approve action của mình). Phức tạp nhưng phần execute action chưa implement (TODO comment). Hiện là dead code.

→ Đơn giản hóa: 1 admin approve/reject, log đầy đủ. Implement 2-approval khi team admin > 2 người.

✕ user_type = 'user' trong enum

Enum user_type có giá trị 'user' nhưng Register.tsx không có option này, authSchema cũng không allow. Không bao giờ được assign. Gây confusion khi đọc code.

→ Remove 'user' khỏi enum, chỉ giữ: athlete | trainer | gym_owner | admin. Cần migration DB nếu có data cũ.
